6 How To Fix This

✅ Option A: Delete Stale File Records (Recommended)

The files table records for Shopify images serve no purpose since products.images[] already has the current URLs. Delete the stale records:

-- Delete file records that point to inaccessible CDN URLs
DELETE FROM product_media WHERE file_id IN (
  SELECT id FROM files WHERE file_size = -1
);

DELETE FROM files WHERE file_size = -1;

✅ Option B: Stop Creating File Records for Shopify Images

Long-term fix: Don't create files table records for Shopify CDN images. They're managed by Shopify, not by your app. Only use the files table for locally uploaded files.

  • Modify Shopify import service to skip file record creation
  • Use products.images[] directly for displaying images
  • Keep files table only for manual uploads

💡 Current Status

The 80 files are already marked with file_size = -1, so they won't produce warnings anymore. But the underlying architectural issue (duplicate image storage) remains and will cause more stale records over time.
