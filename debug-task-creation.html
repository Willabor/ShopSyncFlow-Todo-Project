📋 Debug Checklist - Follow These Steps:

1️⃣ Open Browser Developer Tools

  • Press F12 or right-click → Inspect
  • Go to Console tab
  • Clear any existing logs

2️⃣ Try Creating a Task

  • Open application at localhost:5000
  • Click "New Product" button
  • Fill in the form with test data
  • Submit the form

3️⃣ Check Console Logs

You should see these logs in order:

Frontend logs:
Submitting product data: { title: "...", vendor: "...", ... } Submitting task data: { orderNumber: "...", receivedDate: "...", ... }
Backend logs (check terminal):
Creating product with data: { product: {...}, task: {...} } Parsed productData: {...} Parsed taskData: {...}

4️⃣ If Still Getting 400 Error

Look for validation error details:

In browser console: Check Network tab for response details
In terminal: Look for "Validation errors:" with detailed field errors

🔍 What the Logs Will Tell Us:

Frontend Data Validation:
• Are required fields (title, vendor) filled?
• Is receivedDate formatted as ISO string?
• Are empty strings being sent as empty strings or null?
Backend Schema Validation:
• Which specific fields are failing validation?
• Are the data types correct (string vs Date)?
• Are required fields missing?
Common Issues to Look For:
• Missing vendor selection
• Empty product title
• Invalid date format
• Wrong data structure
